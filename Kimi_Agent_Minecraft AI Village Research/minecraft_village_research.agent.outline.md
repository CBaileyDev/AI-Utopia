# Multi-Agent Minecraft AI Village: Comprehensive Research Report

## Executive Summary
### Key Findings
#### The three-tier architecture (LLM Planner → Goal Specification → Per-Role RL Policies) is optimal but unimplemented in any existing project — a novel contribution opportunity
#### Py4J via UnionClef outperforms the user's proposed gRPC bridge for single-machine deployment; MARLlib should be dropped in favor of direct Ray RLlib
#### Hardware (9800X3D/64GB/RTX 4080) supports 4-6 parallel server instances — the bottleneck is software architecture, not compute
#### Bedrock support imposes a server-side-only constraint that eliminates client-side mod observation sources; this must be decided before coding
#### The practical path is single gatherer first, then sequential role additions — not all 4 roles at once
